title: "Data Model"
description: "Domain entities (Value Objects), their fields, and the relationships between them."
---

Every database table is represented by a plain Java bean in the `eu.esa.gemis.vo` package. There are no ORM annotations; field mapping is performed manually inside each JDBC DAO implementation.

## Entity Relationships

```
CuerpoCeleste ◄──────── Mision ──────── Nave
                           ▲
                           │  (via TripulanteMision)
                        Tripulante

CuerpoCeleste ◄──────── Base ◄──────── GeneradorPrincipal
```

- **`Mision`** holds object references to `Nave` and `CuerpoCeleste` (the destination body).
- **`TripulanteMision`** is the join entity that links `Tripulante` to `Mision` and stores the crew member's `rol` on that mission.
- **`Base`** is located on a `CuerpoCeleste`.
- **`GeneradorPrincipal`** belongs to a `Base`.

## Relationship Summary

| Relationship | Cardinality | How modelled |
|---|---|---|
| `Mision` → `Nave` | Many-to-one | `Mision.nave` (`Nave` reference) |
| `Mision` → `CuerpoCeleste` | Many-to-one | `Mision.cuerpoCeleste` (`CuerpoCeleste` reference) |
| `TripulanteMision` → `Mision` | Many-to-one | `TripulanteMision.mision` |
| `TripulanteMision` → `Tripulante` | Many-to-one | `TripulanteMision.tripulante` |
| `Base` → `CuerpoCeleste` | Many-to-one | `Base.cuerpoCeleste` |
| `GeneradorPrincipal` → `Base` | Many-to-one | `GeneradorPrincipal.base` |
