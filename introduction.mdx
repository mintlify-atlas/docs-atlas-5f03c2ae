# Agencia ESA — GEMIS

**GEMIS** (Gestión de Misiones Espaciales) is a Java console application built for the European Space Agency (ESA) that manages space missions, crew members, spacecraft, and celestial bodies through a MySQL database using the JDBC DAO pattern.

The project was developed as a 2nd-year DAM (Desarrollo de Aplicaciones Multiplataforma) exam exercise at IES Virgen de la Paloma.

## What GEMIS does

GEMIS provides three core operations accessible through an interactive console menu:

## Core design principles

| Principle | Implementation |
|-----------|---------------|
| Separation of concerns | DAO interfaces (`IMisionDAO`, `ITripulanteDAO`, …) decouple business logic from persistence |
| Single connection point | `GestorConexionJDBC` centralises all JDBC connection management |
| External configuration | `aplicacion.properties` controls DB URL, credentials, and file paths |
| Typed error handling | `AgenciaEsaException` carries an error code and the originating class |

## Key entities

- **Mision** — a space mission with name, dates, status, budget, spacecraft reference, and celestial body reference
- **Tripulante** — a crew member with ESA ID, specialty, experience level, and join date
- **Nave** — a spacecraft with a code and crew capacity
- **CuerpoCeleste** — a celestial body (planet, moon, etc.) with name and type
- **TripulanteMision** — the many-to-many relationship between crew and missions
- **Base** / **GeneradorPrincipal** — supporting entities for space stations and power generators
